# Pump-Up 私人健身App 迭代规划

## 用户目标
- 私人健身App，包含丰富的 AI 功能
- 足够好用（工具化）
- AI 服务: **OpenAI API**
- 重点 AI 功能: **运动计划生成、聊天教练、健康分析**
- 饮食功能: **必须有**

## 项目现状概览

**已完成的核心功能：**
- 用户认证系统（注册/登录）
- 运动模板库（6个预设 + 会话追踪）
- 冥想课程库（6个预设 + 历史记录）
- HealthKit 集成（步数、心率、睡眠、体重）
- 成就系统（6种成就）
- 推送通知
- 用户统计数据

## 迭代路线图（按优先级排序）

### Phase 1: AI 智能核心 ⭐⭐⭐⭐⭐

#### 1.1 AI 运动计划生成器
- **功能**: 根据用户目标、身体数据、可用时间自动生成个性化训练计划
- **技术**: OpenAI GPT-4 API
- **后端实现**:
  ```
  新增文件: internal/service/ai/aiImpl/workoutPlan.go
  端点: POST /v1/ai/workout-plan
  ```
- **iOS 实现**:
  ```
  新增文件: Views/AI/AIWorkoutPlanView.swift
  ```
- **用户流程**:
  1. 选择目标（增肌/减脂/塑形/维持）
  2. 设置每周可训练天数（3-6天）
  3. 设置每次时长（30/45/60分钟）
  4. 设置器材条件（家庭/健身房）
  5. AI 生成完整周计划 + 每日详细动作
- **数据结构**: 计划可保存，支持重新生成

#### 1.2 AI 聊天教练
- **功能**: 24/7 智能健身问答助手
- **技术**: OpenAI GPT-4 + 流式响应
- **后端实现**:
  ```
  新增文件: internal/service/ai/aiImpl/chat.go
  端点: POST /v1/ai/chat (支持流式 SSE)
  模型: ChatHistory (存储对话历史)
  ```
- **iOS 实现**:
  ```
  新增文件: Views/AI/AIChatView.swift
  组件: 聊天气泡、流式文字动画、快捷问题
  ```
- **能力范围**:
  - 动作指导（配合已有动作库）
  - 饮食问答
  - 训练问题解答
  - 动力激励
- **上下文**: 自动注入用户身体数据、最近运动记录

#### 1.3 AI 健康分析助手
- **功能**: 分析用户健康数据趋势，提供洞察和建议
- **技术**: OpenAI GPT-4 + 用户历史数据聚合
- **后端实现**:
  ```
  新增文件: internal/service/ai/aiImpl/healthInsights.go
  端点: GET /v1/ai/health-insights
  ```
- **分析维度**:
  - 睡眠质量趋势（过去7天）
  - 运动恢复建议
  - 心率变化分析
  - 活动量评估
  - 体重趋势预测
- **iOS**: 健康仪表盘增加 "AI 洞察" 卡片

### Phase 2: 饮食与营养 ⭐⭐⭐⭐⭐

#### 2.1 饮食记录系统
- **功能**: 记录每日饮食摄入
- **后端实现**:
  ```
  新增模型: internal/model/meal.go
  - Food (食物库: 名称、单位卡路里、蛋白质、碳水、脂肪)
  - MealLog (餐次记录: 用户ID、餐次类型、食物列表、时间)

  新增服务: internal/service/nutrition/
  端点:
  - GET /v1/nutrition/foods (搜索食物库)
  - POST /v1/nutrition/meals (记录一餐)
  - GET /v1/nutrition/today (今日饮食)
  - GET /v1/nutrition/history (饮食历史)
  - GET /v1/nutrition/stats (营养统计)
  ```
- **iOS 实现**:
  ```
  新增文件: Views/Nutrition/NutritionView.swift
  组件: 食物搜索、份量选择、餐次选择
  ```
- **餐次类型**: 早餐、午餐、晚餐、加餐

#### 2.2 AI 饮食建议
- **功能**: 根据训练计划和目标生成个性化饮食方案
- **后端实现**:
  ```
  新增文件: internal/service/ai/aiImpl/nutritionPlan.go
  端点: POST /v1/ai/nutrition-plan
  ```
- **输入**: 用户目标、体重、活动量、饮食偏好
- **输出**:
  - 每日卡路里目标 (TDEE 计算)
  - 宏量营养素分配 (蛋白质/碳水/脂肪)
  - 每餐建议配比
  - 示例餐单

#### 2.3 卡路里平衡追踪
- **功能**: 摄入 vs 消耗可视化
- **iOS 实现**:
  ```
  组件: CalorieBalanceView.swift
  - 环形进度图 (摄入/目标)
  - 消耗分解 (基础代谢+运动)
  - 每日盈亏显示
  ```

#### 2.4 食物库管理
- **预置数据**: 常见中式食物 (米饭、面条、蔬菜、肉类等)
- **用户自定义**: 支持添加自定义食物
- **快速记录**: 最近食物、收藏食物

## 技术实现要点

### OpenAI API 集成

**后端配置** (`config/app_dev.yml`):
```yaml
openai:
  api_key: ${OPENAI_API_KEY}
  model: gpt-4
  max_tokens: 2000
```

**Go 依赖**:
```go
// go.mod
github.com/sashabaranov/go-openai v1.x
```

**服务架构**:
```
internal/service/ai/
├── service.go          # 路由注册
├── aiImpl/
│   ├── client.go       # OpenAI 客户端单例
│   ├── workoutPlan.go  # 运动计划生成
```

# 高级差异化功能设计

> 以下功能是与其他健身App形成差异化的核心竞争力，解决用户真实痛点

## 竞品分析 - 功能盲区

| 维度 | Keep | Nike TC | MyFitnessPal | 我们的机会 |
|------|------|---------|--------------|-----------|
| 训练指导 | 标准化课程 | 明星教练 | 无 | **实时动作纠正** |
| 饮食管理 | 基础记录 | 无 | 详细但繁琐 | **情绪化饮食支持** |
| 个性化 | 有限推荐 | 静态计划 | 无 | **生理周期+情绪状态动态调整** |
| 社交 | 广场式(焦虑) | 无 | 无 | **无压力陪伴式社交** |
| 激励 | 打卡勋章 | 成就系统 | 连续记录 | **内在动机培养** |

## 用户核心痛点深挖

| 表面需求 | 真实痛点 | 解决方案 |
|----------|----------|----------|
| "我要减肥" | 我要感觉更好 | 身体对话日记 |
| "需要训练计划" | 告诉我今天该干啥 | AI身体状态雷达 |
| "记录卡路里" | 太繁琐，情绪化进食 | 拍照+情绪记录 |
| "想要社交" | 不想被比较，要陪伴 | 树洞伙伴匿名社交 |
| "坚持不下去" | 看不到效果，挫败感 | 小胜利里程碑系统 |

## 差异化功能详细设计

### 功能 D1: AI 身体状态雷达 ⭐⭐⭐⭐⭐

**解决痛点**: 不知道今天适合练什么强度、什么部位

**功能描述**:
- 每日晨起 3 分钟交互式评估
- 结合 HealthKit 数据（HRV、静息心率、睡眠）
- 生成 6 维可视化"身体雷达图"
- AI 推荐当日最优训练方案

**6 个评估维度**:
1. 体能储备
2. 肌肉恢复
3. 心肺状态
4. 柔韧性
5. 情绪能量
6. 激素水平（女性生理周期）

**后端实现**:
```
端点: POST /v1/body-radar/assess
模型: internal/model/bodyRadar.go

请求参数:
- sleep_quality: 1-5
- muscle_ache: ["legs", "shoulders"]
- emotion_score: 1-10
- stress_level: 1-5
- menstrual_phase: "follicular"/"ovulation"/"luteal"/"menstrual"

响应:
- dimensions: {体能:85, 恢复:60, 心肺:90, ...}
- recommendation: {type:"yoga", intensity:"light", duration:30, reason:"..."}
```

**iOS 实现**:
```
Views/BodyRadar/BodyRadarView.swift
Views/BodyRadar/RadarChartView.swift (雷达图组件)
```

### 功能 D2: 情绪驱动饮食记录 ⭐⭐⭐⭐⭐

**解决痛点**: 传统卡路里记录太繁琐，忽视情绪化进食

**功能描述**:
- 拍照 + 语音记录饮食，AI 自动识别
- **核心创新**: 记录进食时的情绪标签
- 生成"情绪-饮食关联报告"
- 识别暴食触发器，提供替代建议

**技术方案**:
```
端点: POST /v1/nutrition/emotion-log

请求:
- image_base64: 食物照片
- emotion: "happy"/"stress"/"anxiety"/"bored"
- meal_time: "breakfast"/"lunch"/"dinner"/"snack"

响应:
- food_items: [{name, calories, protein, carbs, fat}]
- emotion_trigger: "检测到压力时段易摄入高糖零食"
- suggestion: "推荐用15分钟散步替代下午茶"
```

**AI 视觉识别**: 使用 OpenAI Vision API 识别食物

**iOS 界面**:
```
[拍照界面] → [AI识别结果] → [选择情绪] → [记录成功]

周报告展示:
- 压力时段饮食分析
- 情绪-食物关联热力图
- 个性化建议
```

### 功能 D3: 树洞伙伴 - 无压力微社交 ⭐⭐⭐⭐

**解决痛点**: 广场式社交引发焦虑，但又需要情感支持

**功能描述**:
- 匿名匹配"树洞伙伴"（相似目标、相似阶段）
- **仅能看到对方的"感受日记"，看不到成绩数据**
- 互相发送"今日能量"（鼓励话 + 虚拟礼物）
- 阶段性可选择"破冰见面"

**核心设计原则**:
- 看感受不看成绩 → 避免攀比
- 匿名机制 → 降低心理负担
- 并肩前行 → 非竞争陪伴

**后端实现**:
```
模型:
- BuddyConnection: {user1, user2, status, anonymous}
- FeelingsJournal: {user_id, content, emotion, share_scope}
- EnergyGift: {sender, receiver, message, gift_type}

端点:
- POST /v1/buddy/match (匹配伙伴)
- POST /v1/buddy/journal (发布感受日记)
- POST /v1/buddy/energy (发送能量)
```

### 功能 D4: 生理周期训练计划（女性专属）⭐⭐⭐⭐⭐

**解决痛点**: 女性用户不知道生理期如何科学训练

**四阶段差异化方案**:

| 阶段 | 天数 | 推荐训练 | 避免 | 营养重点 |
|------|------|----------|------|----------|
| 月经期 | 1-5天 | 低强度瑜伽、散步 | HIIT、重量训练 | 补铁、维生素B |
| 卵泡期 | 6-14天 | **力量训练黄金期**、HIIT | 无 | 高蛋白支持肌肉合成 |
| 排卵期 | 14-16天 | 体能巅峰，挑战重量 | 无 | 复合碳水补充能量 |
| 黄体期 | 17-28天 | 适度有氧、瑜伽 | 过度疲劳 | 控制糖分摄入 |

**后端实现**:
```
模型: internal/model/menstrualCycle.go
- last_period_start
- cycle_length
- current_phase
- symptoms

端点: GET /v1/cycle/today-plan
响应:
- phase: "卵泡期"
- phase_day: 8
- recommended_types: ["strength", "hiit"]
- intensity_level: "high"
- explanation: "雌激素上升期，肌肉合成效率提高30%"
- avoid_exercises: []
- nutrition: {focus: ["蛋白质"], recommendation: "..."}
```

**iOS 实现**:
```
Views/Cycle/CycleCalendarView.swift (周期日历)
Views/Cycle/CycleRingView.swift (周期环形图)
```

### 功能 D5: 身体对话日记 ⭐⭐⭐⭐

**解决痛点**: 用户只关注体重数字，忽视身体质性变化

**功能描述**:
- 每周引导记录非数字化的身体变化
- AI 分析日记生成"非体重成就单"
- 对抗"体重焦虑"，建立内在成就感

**记录维度**:
- 精力充沛程度 (⭐1-5)
- 体态改善描述 ("肩膀不那么含了")
- 情绪稳定性 ("焦虑发作减少")
- 日常活动能力 ("能轻松抱起孩子")

**交互示例**:
```
[本周身体对话时间]

虽然体重只减了0.5kg，但你获得了：
✅ 睡眠质量提升 20%
✅ 日常活力指数 +15
✅ 3 个身体正向反馈

记住：健康是感受，不仅是数字！
```

### 功能 D6: 智能退阶系统 ⭐⭐⭐⭐

**解决痛点**: 计划太难完成不了产生挫败感

**功能描述**:
- 训练中实时监测心率、动作完成度
- 当检测到"跟不上"时，**立即弹出选项**
- **关键创新**: 不惩罚降级，反而奖励"聪明的调整"

**触发条件**:
- 心率超过 85% 最大心率
- 动作质量评分连续 < 60
- 用户主观反馈"太累了"

**降级选项**:
1. 降低强度 → 切换简化版动作
2. 短暂休息 → 休息 30 秒后继续
3. 更换训练 → 切换到备选方案

**奖励机制**:
```
选择降级后：
🎖️ 获得"倾听身体"勋章
"聪明的调整也是训练的一部分！"
```

### 功能 D7: 碎片时间训练库 ⭐⭐⭐⭐

**解决痛点**: 忙碌用户找不到完整时间训练

**分类体系**:

| 时长 | 场景 | 示例 |
